The Problem

When a service is deprecated,
its soul doesn't leave production.

Every deprecated service leaves behind invisible behavioral contracts — undocumented expectations that downstream systems silently depend on. Tests can't see them. Logs don't record them. And when the replacement service breaks them, production burns quietly.

" It passed all tests. It looked fine. Two weeks later, payments were silently dropping with no error in any dashboard.
01

Tests can't see behavior

Unit tests verify logic, not temporal contracts. Latency bands, error frequency, side effects, and cache behaviors are invisible to your test suite.

02

Replacements look compliant

The new service passes all validation. Integration tests pass. But it doesn't preserve the exact behavioral fingerprint that 23 downstream services depend on.

03

By the time you know, it's live

Contract breaks surface days after deployment, in symptoms that seem unrelated. Root cause is invisible without a forensic record of what used to be true.

How It Works

Four steps from registration
to ghost detection

01

Register

Point Karma at your deprecated service endpoint before migration begins. Registration takes under 30 seconds.

02

Observe & Learn

Karma watches 24–72 hours of live traffic. Gemini 2.5 extracts implicit behavioral contracts invisible to tests.

03

Haunt

Ghost deployed against the replacement. Continuous behavioral comparison runs in real-time via Dynatrace MCP.

04

Alert & Remediate

Detailed ghost reports fire with Dynatrace evidence, root cause analysis, severity scores, and AI remediation paths.

Capabilities

AI Contract Learning

Gemini 2.5 Flash observes live traffic and extracts implicit behavioral contracts that no developer ever wrote down — latency percentiles, error rate bands, side effects, and temporal patterns invisible to any test suite.

Powered by Gemini 2.5

Ghost Detection

Continuous comparison against learned contracts. Every request to the replacement service is measured against its predecessor's behavioral fingerprint.

Dynatrace Native

Built on the Dynatrace MCP Server. Forensic evidence pulled directly from your existing observability infrastructure — zero additional instrumentation.

MCP Integration

Multi-Agent Orchestration

Google ADK v1.0 coordinates four specialized agents: Observer, Extractor, Comparator, and Alerter — each owning one phase of the detection pipeline.

Live Ghost Alerts

SSE-powered real-time alerts stream to your dashboard within seconds. Each report includes contract violation details, severity classification, Dynatrace trace IDs, and AI-generated remediation suggestions.
